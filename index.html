Visión General del Sistema

La plataforma ProManage se descompone en dominios funcionales claros, cada uno diseñado como un servicio independiente para maximizar la agilidad y la escalabilidad. A continuación se presentan los componentes clave que forman el núcleo de la aplicación.

Gestión de Usuarios

Maneja el registro, autenticación, perfiles de usuario, roles y permisos de acceso.

Gestión de Proyectos

Permite la creación, modificación, visualización y seguimiento del estado de los proyectos.

Gestión de Presupuestos

Se encarga de la asignación y el control del presupuesto para cada proyecto individual.

Gestión de Archivos

Facilita la carga, descarga y asociación de documentos y archivos relevantes a los proyectos.

Gestión de Tareas y Plazos

Define fechas de entrega, gestiona tareas y realiza el seguimiento de los plazos.

Notificaciones

Envía alertas y notificaciones sobre cambios importantes, plazos y comentarios en la plataforma.

Modelo Arquitectónico

La elección de la arquitectura es una decisión fundamental que impacta la escalabilidad, resiliencia y velocidad de desarrollo. Hemos optado por un enfoque moderno y flexible.

Decisión: Arquitectura de Microservicios

Se elige una arquitectura basada en microservicios por su superioridad en escalabilidad, resiliencia y flexibilidad frente a un modelo monolítico. Esto permite que cada componente del sistema evolucione y escale de forma independiente.

Estrategias de Resiliencia

🏙️🏢

Despliegue Multi-AZ

El sistema opera en múltiples centros de datos para sobrevivir a fallos geográficos.

❤️‍🩹

Health Checks

Kubernetes monitoriza y reinicia automáticamente contenedores que no responden.

🔄🔄🔄

Múltiples Réplicas

Cada servicio corre con varias instancias para garantizar que no haya puntos únicos de fallo.

Contenedorización y Orquestación

Utilizamos contenedores para empaquetar y aislar nuestras aplicaciones, y un orquestador para gestionar su ciclo de vida a gran escala de forma automatizada.

Contenerización

Herramienta: Docker

El estándar de la industria para crear imágenes ligeras y portables de nuestros microservicios.

Orquestación

Herramienta: Kubernetes

Automatiza el despliegue, escalado y la gestión de los contenedores a través de un cluster.

Registro de Imágenes

Herramienta: AWS ECR / GCR

Un repositorio privado, seguro y de alta disponibilidad para almacenar nuestras imágenes de Docker.

Flujo de Despliegue Continuo (CI/CD)

Un pipeline automatizado lleva el código desde el repositorio hasta producción de forma segura y eficiente.

1. Código en Git

El desarrollador sube cambios

2. Pipeline CI/CD

Se ejecutan tests y builds

3. Push a Registro

La imagen Docker se guarda en ECR

4. Deploy en K8s

Se actualiza el servicio en producción
